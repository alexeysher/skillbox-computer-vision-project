# Skillbox. Дипломная работа по компьютерному зрению. Распознавание эмоций человека

https://www.kaggle.com/c/skillbox-computer-vision-project

В рамках данной дипломной работы был разработан ноутбук, с помощью которого можно создать готовую к использованию модель распознавания эмоций по изображению лица человека. Ноутбук позволяет создать модель одно из следующих типов:
1. Модель, которая может по выражению лица человека пресказать эмоцию, которую они испытывает.
2. Модель, которая может по выражению лица человека предсказывать уровни валентности *(англ. valence)* и интенсивности *(англ. arousal)* эмоции, которую он испытывает, по шкале от -1 до +1.

Тип создаваемой модели определяется способом описания эмоций, которые модель должна обрабатывать, в настройках проекта. Если эмоции описываются простым списком или кортежем из их названий, то создается первый тип модели. Если описание эмоций представлено словарем, в котором каждой эмоции сопоставляется пара значений, характеризирующих типичные уровни валентности и интенсивности этой эмоции, то создается модель второго типа.

## Архитектура модели

![Model drawio](https://user-images.githubusercontent.com/107345313/199323329-ce7b4696-0d32-4a85-8886-ac058d116cde.svg)

Базовая модель обучена предсказывать какой объект из датасета [ImageNet](https://www.image-net.org/) представлен на изображении на основании его признаков. За извлечение признаков отвечает свёрчтоная часть модели. За индентификацию объекта отвечает последний, находящийся как бы вверху модели *(англ. on top)*, полносвязный слой с функцией активации типа [SoftMax](https://en.wikipedia.org/wiki/Softmax_function). Поскольку строящаяся модель решает совершенно другую задачу, то этот полносвязный слой (англ. dense layer) в модели необходимо заменить. Т.е. из базовой модели используются только свёрточная часть базовой модели. Для получения на выходе одномерного верктора признаков к последнему свёторчному слою добавляется пуллинг слой. Тип пуллинга (максимальный или средний) задается в настройках проекта (параметр `BASE_MODEL_POOLINGS`). 

В общем случае к слою пуллинга добавляется один или несколько полносвязных слоев с функцией активации типа [ReLU](https://en.wikipedia.org/wiki/Rectifier_(neural_networks)). Кроме того для обеспечения регуляризации в процессе обучения перед каждым таким слоем располагается слой исключения (англ. dropout layer). Такой слой в процессе обучения случайным образом обнуляет заданную часть входных нейронов. При этом оставшиеся входные нейроны пропроционально увеличиваются так, чтобы сумма всех нейронов оставалась неизменной.

Пользователь может задать различные варианты количества таких слоев, варианты количества выходных нейронов в них, а также варианты величины доли исключаемых данных.

При построении модели 1-го типа добавляется ещё один полносвязный слой с функцией активации типа Softmax и количеством выходных нейронов, соответствующих количеству распознаваемых эмоций. В этом случае на выходе модели при инференсе формируется вектор верояностей того какую эмоцию выражает лицо человека на входном изображени.

При построении модели 2-го типа Полносвязная модель представляет собой последовательную модель, состоящую из одного или нескольких полносвязных слоев (Dense layers). Добавляется ещё один полносвязный слой с 2-мя выходными нейронами и функцией активацией типа ReLU с ограничение значения до 2. Т.о. выходные нейроны этого слоя могут принимать значения в диапазоне от 0 до 2. Но так значения valence и arousal должны находиться в диапазоне от -1 до 1, то в модель добавляется ещё один слой, который вычитает 1 из этих значений.

Дополнительно повышения качества работы модели в процессе обучения модель учится не на исходных изображениях, а на их случайным образом преобразованных вариантах. Исходное изображение перед подачей в базовую модель с помощью модели аугментации случайным образом немного поворачивает изображения, изменяет его контраст и яркость, а также зеркально отражает его.

<details><summary>Пример задания вариантов конфигураций полносвязных слоев</summary>
<p>
    
```python
BASE_MODEL_POOLINGS = 'avg' # Тип пулинга на выходе базовой моделей ('avg' - average, 'max' - max)
MODEL_ON_TOP_DENSE_NUMS = [1, 2] # Варианты количества дополнительных полносвязных слоев
MODEL_ON_TOP_DENSE_UNITS = [1024, 2048] # Варианты количества выходных нейронов в дополнительном полносвязном слое
MODEL_ON_TOP_DROPOUT_RATES = [.0, .2] # Варианты доли исключаемых данных перед подачей в полносвязный слой при обучении
```
    
</p>
</details>

При создании модели первого типа в конце модели добавляется полносвязный слой с функцией активации типа [SoftMax](https://en.wikipedia.org/wiki/Softmax_function) и количеством выходных нейронов равным количеству предсказываемых эмоций.
При создании модели второго типа после полносвязных слоев также добавляется полносвязный слой, но только с двумя выходными нейронами и функцией активации типа [ReLU](https://en.wikipedia.org/wiki/Rectifier_(neural_networks)) с ограничением выходных значений до 2. Т.о. на выходе этого слоя формируется пара значений в диапазоне от 0 до 2. Для приведения этих значений к шкале (от -1 до 1) добавляется ещё один слой, в котором из этих значений вычитается 1. Т.о. на выходе модели мы получаем пару значений в диапазоне от -1 до 1.

## Пайпланы
Процесс создания системы разделен на три независимых стадии (пайплайна), которые выполняются последовательно:
1. Предобработка изображений (`IMAGE_PREPROCESSING_PIPELINE`).
2. Сбор информации о базовых моделях в Keras Applications (`KERAS_BASE_MODELS_PROCESSING_PIPELINE`).
3. Создание модели (`MODEL_BUILDING_PIPELINE`).


Каждый пайплайн в свою очередь состоит из нескольких этапов, которые также должны выполняться последовательно. Любой этап на усмотрение пользователя может выполняться как на локальном компьютере в [JupyterNotebook](https://jupyter.org/) или [JupyterLab](https://jupyterlab.readthedocs.io/en/stable/index.html), так и удаленно в [Google Colab](https://colab.research.google.com/). Платформа для выполнения этапа задается в его настройках (параметр `platform`). Такая опция позволяет выполнять этапы на наиболее подходящей платформе. Например, этапы, которые не требуют использования GPU целесообразнее выполнять на локальном компьютере. И наоборот, в случае отсутствия на локальном компьютере подходящего для машинного обучения GPU, этапы, в рамках выполнения которых производится работа с базовой моделью, целесообразно выполнять в [Google Colab](https://colab.research.google.com/).

Для того чтобы облегчить переключение между платформами, все данные сохраняются в едином облачном хранилище [Google Диск](https://drive.google.com/). ### 1. Предобработка изображений (`IMAGE_PREPROCESSING_PIPELINE`).
Данный пайплайн предназначен для подготовки тренировочных и тестовых датасетов. 

Извелечение лиц из оригинальных датасетов осуществляется с помощью предобученного детектора лиц [MTCNN](https://github.com/ipazc/mtcnn), реализованной Iván de Paz Centeno <ipazc@unileon.es>. Детектор построен на базе многозадачной сверточной нейронной сети *(англ. Multi-task Cascaded Convolutional Network)* описанной в статье Kaipeng Zhang, Zhanpeng Zhang, Zhifeng Li и Yu Qiao "[Joint face detection and alignment using multitask cascaded convolutional networks](https://arxiv.org/abs/1604.02878)".


#### 1.1. Извлечение изображений лиц из тренировочного датасета (`train_face_extraction`)
Данный этап преследует две цели:
1. Убрать из изображений лишние детали, которые могут "путать" модель.
2. Исключить из датасета изображения на которых нет изображений лиц, либо изображено несколько людей, либо лица людей показаны не полностью.

В результате выполнения этапа создается датасет извлеченных изображений лиц, zip-архив которого сохраняется в общем хранилище. Путь к датасету на локальной платформе и соответственно имя архива определяется параметром `path` в настройках этапа.

В целях оптимального использования вычислительных ресурсов платформы и сокращения времени выполнения пайплайна все этапы выполняются с использованием механизма параллельных вычислений, реализуемого с помощью пакета [IPython Parallel](https://ipyparallel.readthedocs.io/en/latest/). Количество паралллельных движков *(англ. engines)*.

#### 1.2. Извлечение изображений лиц из тестового датасета (`test_face_extraction`)
Целью данного этапа является только извлечение лиц из тестового датасета (отсечение лишней информации). Считается, что тестовые данные оприори корретны, т.е. все изображения содержат полное изображение хотя бы одного лица. Поэтому если на хотя бы одном исходном изображении лицо не обнаружется, то этап считается не выполненным и нужно скорретировать настройки детектора и запустить его выполнение повторно.

#### 1.3. Балансировка тренировочного датасета (`train_balancing`)
Целью данного этапа является выравнивание количества изображений лиц для каждой эмоции. Это необходимо для того, чтобы снизить перекос в способности модели распознавании моделью разных эмоций. Выравнивание достигается за счёт увеличения количества изображений до максимального размера в исходном датасета. Добавляемые лица выбираются случайным образом.

#### 2.3. Выбор базовой модели (`base_model_selection`)
Целью данного этапа является выбор базовой модели, которая будет использована при построении результирующей модели в следующем пайплайне. Выбор происходит на основании наилучшего сочетания быстродействия и точности модели, показанной на валидационном датасете ImageNet, с учетом весов этих метрик, заданных пользователем (параметры `inference_time_weight` и `top1_accuracy_weight` соответственно).

Кроме вышеуказанных метрик на выбор базовой модели влияет её размер. Пользователь может ограничть круг рассматриваемых моделей, задав максимально допустимый размер модели в МБ (параметр `BASE_MODEL_MAX_SIZE` в разеделе "Основные настройки").

Для каждой отобранной модели начисляются очки точности и очки быстродействия. Очки за точность начисляются согласно следующему правилу:

$$Sa_i = {a_i - min(a_1, ..., a_n)\over max(a_1, ..., a_n) - min(a_1, ..., a_n)},$$  
где&nbsp; $a_i$ - точность оцениваемой модели;  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $a_1, ..., a_n$ - точность оцениваемых моделей.

$$St_i = {max(t_1, ..., t_n) - t_i \over max(t_1, ..., t_n) - min(t_1, ..., t_n)}$$
где&nbsp; $t_i$ - время инференса оцениваемой модели;  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $t_1, ..., t_n$ - время инференса оцениваемых моделей.

$$S_i = Sa_i Wa + St_i Wt $$
где&nbsp; $Sa_i$ - очка за точность оцениваемой модели;  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $St_i$ - очки за быстродействие оцениваемой модели;  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $Wa$ - вес оценки за точность;  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; $Wt$ - вес оценки за быстродействие.

- `base_model_name` - название базовой модели,
- `top1_accuracy` - точность базовой модели на валидационном датасете ImageNet,
- `size` - размер базовой модели в МБ,
- `colab_inference_time` - измеренное на предыдущем этапе время инференса в секундах,
- `top1_accuracy_score` - очки за точность,
- `colab_inference_time_score` - очки за быстродействие,
- `weighted_score` - взвешенные очки за точность и быстродействие,
- `rank` - ранг соответствия модели.

Для дальгейшей работы выбирается модель, имеющая ранг #1, т.е. лучшую взвешенную оценку за точность и быстродействие. 
Её название и основные характеристики сохраняются в результирующий csv-файл со следующими полями:
- `base_model_name` - название;
- `image_size` - размер входного изображения;
- `feature_size` - размер вектора признаков.
